## What This Is

Async Yandex Passport (mobile) auth library extracted from Music Assistant's Yandex Music provider. Covers the full token-derivation graph (QR login, x_token, music_token, session cookies, Quasar CSRF, Glagol device token) used by four MA providers: music, ynison, smarthome, station.

Published to PyPI as `ya-passport-auth`. Import as `ya_passport_auth`. MIT licensed.

## Architecture

**`PassportClient`** (`client.py`) is the single public facade. It owns an `aiohttp.ClientSession` with a dedicated `CookieJar` and wires together:

- **`SafeHttpClient`** (`http.py`) — every HTTP call goes through this. Enforces host allow-list, response size caps (1 MiB JSON / 2 MiB HTML), rate limiting, disabled redirects, and wraps all errors into the exception hierarchy.
- **`AsyncMinDelayLimiter`** (`rate_limit.py`) — serializes requests with a minimum delay between them. Injectable clock for testing.
- **Flow modules** (`flows/`) — each implements one Yandex API interaction. They receive a `SafeHttpClient` and never make raw HTTP calls.

Flow call graph for QR login:
```
PassportClient.poll_qr_until_confirmed
  → QrLoginFlow.get_qr          (CSRF extract → submit → QrSession)
  → QrLoginFlow.check_status    (poll loop)
  → _complete_auth               (shared token exchange)
    → exchange_cookies_for_x_token  (session cookies → x_token)
    → exchange_x_token_for_music_token (x_token → music_token)
    → AccountInfoFetcher.fetch    (x_token → uid/login, graceful failure)
  → returns Credentials
```

Flow call graph for cookie login:
```
PassportClient.login_cookies
  → CookieLoginFlow.login        (raw cookies → x_token)
  → exchange_x_token_for_music_token → Credentials
```

**Security invariants** (see SECURITY.md for full threat model T1-T14):
- `SecretStr` wraps all tokens — redacted in repr/str/format/pickle. Never pass raw token strings through exception messages or logs.
- `RedactingFilter` (`logging.py`) scrubs `OAuth \S+` headers and 32+ hex runs from all log output.
- `_check_host()` validates every request URL against `ClientConfig.allowed_hosts` (frozenset exact match via `yarl.URL.host`).
- `Credentials` is frozen+slotted — no `__dict__`, no pickle, no mutation.

## Testing Patterns

- **All HTTP mocking** uses `aioresponses`. No live network calls in tests.
- **Fixtures** in `tests/fixtures/` are sanitized HTML/JSON captures. Test tokens match `test-token-*` / `test-*-0123...` patterns.
- **`pytest-asyncio` auto mode** — async test functions work without `@pytest.mark.asyncio`.
- **Hypothesis** fuzzes CSRF regex patterns (`test_csrf_fuzz.py`).
- Tests are organized: `unit/` (primitives), `flows/` (per-flow with mocked transport), `integration/` (PassportClient facade end-to-end).

## Key Constraints

- Python >=3.12 only. Uses `X | Y` union syntax, `frozen=True, slots=True` dataclasses.
- `mypy --strict` with `disallow_any_explicit = true` — use `dict[str, object]` not `dict[str, Any]`.
- ruff line length 100, Google docstring convention.
- `constants.py` contains public OAuth client IDs from Yandex Android apps — these are intentionally committed (not secrets). Per-file `S105` ignore.
- `_version.py` is generated by hatch-vcs at build time. Source checkouts use `try/except ImportError` fallback.
